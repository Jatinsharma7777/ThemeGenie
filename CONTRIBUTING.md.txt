# Contributing to ThemeGenie

Thank you for your interest in contributing to ThemeGenie.

We welcome contributions that improve the extension, fix bugs, enhance compatibility, or introduce useful features. Before submitting a contribution, please read the guidelines below.

## Getting Started

1. Fork the repository.
2. Clone your fork locally.
3. Create a new branch for your work.

```bash
git checkout -b feature/your-feature-name
```

4. Make your changes.
5. Test your changes before submitting.
6. Commit your work with a clear commit message.
7. Push your branch and open a Pull Request.

## Code Guidelines

* Keep code clean and well documented.
* Use meaningful variable and function names.
* Avoid unnecessary dependencies.
* Follow the existing project structure.
* Test new features before opening a Pull Request.

## Bug Reports

When reporting a bug, please include:

* Browser version
* Operating system
* Steps to reproduce
* Expected behavior
* Actual behavior
* Screenshots (if applicable)

## Feature Requests

Feature requests are welcome.

Please explain:

* The problem you are trying to solve.
* Your proposed solution.
* Any alternatives you considered.

## Pull Requests

Please ensure that:

* Your code builds successfully.
* Existing functionality is not broken.
* New features are documented when necessary.
* Pull requests remain focused on a single feature or fix.

## Code of Conduct

Please be respectful and constructive when interacting with other contributors. We aim to maintain a welcoming and collaborative environment for everyone.

Thank you for helping improve ThemeGenie.
